Промежуточная аттестация
    1. Напишите позитивные и негативные тесты на эндпоинт https://idratherbewriting.com/learnapidoc/assets/files/swagger/#/Current%20Weather%20Data/CurrentWeatherData.

Для отправки запросов понадобится регистрация на сайте (можно указать псевдоним, компанию можно не указывать, нажав на cancel), в личном кабинете будет доступен ключ для авторизации в апи. Инструкция здесь под пунктом “Activity: Get an OpenWeatherMap API key”. После получения ключа, его вставляют как параметр в код appid=ab26a2beceb94848c646c4bcc3554a2b. В сваггере он вставляется сюда:



Будьте внимательны! Апи ключ может активироваться в течение часа, поэтому не откладывайте выполнение задачи до победного. Не забудьте подтвердить регистрацию через письмо на почте.

Для тестов создайте новый проект на грэдл. Можете использовать парсинг метода из сваггера или прописать метод вручную в самих тестах. Для тестов используйте RestAssured или Karate (этот фреймворк советую только тем, кто уже ранее работал с RestAssured и хочет попробовать что-то новое).

Не забывайте переиспользовать код, пишите спецификации для запросов и ответов, так как тестов будет несколько, эндпоинт в них один и тот же и результат тоже будет у некоторых совпадать. На выходе у вас должно получиться не менее 8 тестов, но слишком много тоже не пишите, пользуйтесь техниками тест-дизайна.

Если какой-то тест падает по независящим от вас причинам, поставьте комментарий с пояснением перед этим методом. Так проверяющему будет проще понять, что и где у вас падает, чтобы помочь разобраться.

    2. Подключите allure report и выведите в readme скриншот отчета по всем вашим тестам.
